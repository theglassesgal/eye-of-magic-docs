How the Bridge Works

This system acts as a high-speed, live communication bridge linking Discord directly to running Roblox game servers. Commands triggered within Discord do not talk to Roblox directly; instead, they alter a centralized data queue managed by the backend Python application.

Every running game instance inside your Roblox universe runs a background script that repeatedly checks this web service. When a new command entry appears in the queue matching that universe, the Roblox server downloads the command payload and applies the moderation logic to the player instantly.

Moderation Commands

All moderation interactions must run inside authorized Discord channels using the system prefix.

mg.ban <username> [reason]

Issues a universal ban blocking the targeted player account from entering any server instance associated with the game universe.

mg.unban <username>

Removes a target user from the active global blacklist dataset, enabling them to reconnect cleanly.

mg.kick <username> [reason]

Immediately drops a player's connection from whichever active server instance they currently occupy.

mg.flag <username> [reason]

Appends a specific warning tag to the user configuration file. When this user joins any server instance, a detailed system warning triggers in the administration channel log.

mg.unflag <username>

Clears existing flags and warnings from the selected account profile storage record.

Utility Commands

mg.clear [amount]

Cleans up log channels by removing messages. Specifying no value or an amount greater than 100 triggers an automated room recreation process to completely clear the view instantly.

mg.help

Brings up a clean inline summary menu displaying available commands directly inside the active Discord client interface.

Host Console Commands

These instructions must be typed straight into the terminal workspace hosting the executable script process. They do not operate through Discord channels.

stop

Safely terminates the live hosting thread, releases web binding allocations on port 8080, and breaks the active connection profile link with Discord.

st

Prints a structured matrix directly inside the command line displaying all valid status codes and layout formats.

status <type> <message>

Alters the live activity presence text string tied to the application profile window on Discord.

  • 0 or playing -> Displays as: Playing [Message]
  • 1 or streaming -> Displays as: Streaming [Message]
  • 2 or listening -> Displays as: Listening to [Message]
  • 3 or watching -> Displays as: Watching [Message]
  • 5 or competing -> Displays as: Competing in [Message]
