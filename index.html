Eye of MAGIC

Architecture Overview

This system operates as a real-time, low-latency bridge linking Discord directly to active Roblox server instances. Commands executed within Discord do not interface with Roblox directly; instead, they modify a centralized moderation queue managed by the host application.

Active game instances query this queue on a structured interval. When a pending moderation action is detected, the server instance retrieves the payload and executes the restriction immediately against the target player.

Moderation Directives

Execution requires proper authorization parameters within the Discord environment.

mg.ban <username> [reason]

Issues a persistent universe ban, blocking the specified account from connecting to any active server node.

mg.unban <username>

Purges the target user from the global blacklist, restoring connection privileges.

mg.kick <username> [reason]

Forcefully terminates the user's connection to their current active server node.

mg.flag <username> [reason]

Applies an administrative warning tag. When this account joins any server, a priority alert is broadcasted to the system logs.

mg.unflag <username>

Removes the administrative warning tag from the target profile.

System Utilities

mg.clear [amount]

Purges channel message logs. Executing without parameters, or specifying a value exceeding 100, will trigger an automated channel clone protocol to execute an instant, complete wipe of the environment.

mg.help

Retrieves the localized command reference index within Discord.

Host Terminal Directives

These commands are strictly for the local host terminal running the Python executable. They will not process via Discord.

stop

Initiates a safe shutdown sequence, unbinding the web server from port 8080 and cleanly severing the Discord API connection.

st

Outputs the valid presence status codes to the local terminal display.

status <type> <message>

Modifies the rich presence activity string attached to the application profile.

  • 0 -> Playing [Message]
  • 1 -> Streaming [Message]
  • 2 -> Listening to [Message]
  • 3 -> Watching [Message]
  • 5 -> Competing in [Message]
